# [Frontend] 방 입장 플로우

## 방 생성하기 플로우

```
[Frontend] 방 생성 버튼 클릭
↓
RoomModal.tsx - handleCreateSubmit() 실행
↓
폼 데이터 변환 (CreateRoomFormData → CreateRoomRequest)
  - title → roomName
  - targetBudget (문자열) → targetBudget (숫자)
  - mode (personal/host) → syncMode (FOLLOW/LEAD)
  - category (쉼표 구분 문자열) → interestCategories (배열)
  - purpose → shoppingPurpose
  - participants → headcount
  - minBudget → budgetMin
↓
POST /api/rooms
  - 엔드포인트: `${API_BASE_URL}/api/rooms`
  - 함수: createRoom() (src/entities/room/api/room.ts)
  - 요청 body: {
      "roomName": "함께하는 쇼핑",
      "targetBudget": 500000,
      "syncMode": "FOLLOW",
      "roomMeta": {
        "shoppingPurpose": "친구들과 함께하는 겨울 쇼핑",
        "interestCategories": ["패션", "전자제품", "가구"],
        "headcount": 4,
        "budgetMin": 100000
      }
    }
↓
응답에서 roomId 획득 (ex. 17)
  - response.data.data.roomId
↓
라우팅 이동 (/rooms/17)
  - navigate(`/rooms/${response.roomId}`)
  - DesktopRouter.tsx의 Route path="/rooms/:roomId" 매칭
↓
DesktopVideoChatPage 컴포넌트 렌더링
  - src/pages/desktop/VideoChat/index.tsx
  - useParams()로 roomId 추출 가능 (현재 코드에는 미구현)
↓
WebRTC 시그널링 서버에 roomId로 join
  - POST /api/rooms/:roomId/webrtc/sessions
  - 함수: joinWebrtcSession() (src/entities/room/api/webrtc.ts)
  - 요청 body: {
      "role": "PUBLISHER",
      "data": JSON.stringify({
        "nickname": "...",
        "profileColor": "..."
      })
    }
  - accessToken 필요 (Authorization 헤더)
↓
OpenVidu 세션 초기화 및 연결
  - useOpenViduSession hook 사용 (src/features/video-chat/model/useOpenViduSession.ts)
  - OpenVidu 객체 생성 및 세션 초기화
  - session.connect(sessionInfo.token) 실행
  - PUBLISHER 역할인 경우 Publisher 생성 및 publish
↓
화상채팅 연결 시작
  - isConnected 상태가 true로 변경
  - streamCreated 이벤트로 다른 참여자 스트림 구독
  - 로컬 비디오 스트림 publish 완료
```

## 방 참여하기 플로우

```
[Frontend] 방 참여 버튼 클릭
↓
RoomModal.tsx - handleJoinSubmit() 실행
↓
입장 링크에서 roomCode 추출
  - URL 형식인 경우 마지막 경로 추출
  - 쿼리 파라미터 제거
↓
POST /api/rooms/join
  - 엔드포인트: `${API_BASE_URL}/api/rooms/join`
  - 함수: joinRoom() (src/entities/room/api/room.ts)
  - 요청 body: {
      "roomCode": "549F9C01",
      "nickname": "게스트2"
    }
↓
응답에서 roomId 획득 (ex. 4)
  - response.data.data.roomId
↓
라우팅 이동 (/rooms/4)
  - navigate(`/rooms/${response.roomId}`)
  - DesktopRouter.tsx의 Route path="/rooms/:roomId" 매칭
↓
DesktopVideoChatPage 컴포넌트 렌더링
  - src/pages/desktop/VideoChat/index.tsx
↓
WebRTC 시그널링 서버에 roomId로 join
  - (방 생성 플로우와 동일)
↓
화상채팅 연결 시작
  - (방 생성 플로우와 동일)
```

## 주요 컴포넌트 및 파일

### 방 생성/참여
- **RoomModal.tsx** (`src/widgets/desktop/RoomModal/RoomModal.tsx`)
  - 방 생성/참여 폼 UI 및 제출 처리
- **createRoom()** (`src/entities/room/api/room.ts`)
  - 방 생성 API 호출
- **joinRoom()** (`src/entities/room/api/room.ts`)
  - 방 참여 API 호출

### 라우팅
- **DesktopRouter.tsx** (`src/app/router/DesktopRouter.tsx`)
  - `/rooms/:roomId` 경로 정의
- **DesktopVideoChatPage** (`src/pages/desktop/VideoChat/index.tsx`)
  - 방 입장 후 렌더링되는 메인 페이지

### WebRTC 연결
- **joinWebrtcSession()** (`src/entities/room/api/webrtc.ts`)
  - WebRTC 시그널링 서버 세션 생성 API 호출
- **useOpenViduSession** (`src/features/video-chat/model/useOpenViduSession.ts`)
  - OpenVidu 세션 관리 hook
  - 세션 연결, Publisher/Subscriber 관리

## 참고사항

- Desktop 버전에서는 현재 `DesktopVideoChatPage`에서 `useOpenViduSession`을 직접 사용하지 않음
- Mobile 버전(`src/pages/mobile/video-chat/index.tsx`)에서는 `useOpenViduSession`을 사용하여 WebRTC 연결 구현됨
- Desktop 버전에서도 동일하게 `useParams()`로 `roomId`를 추출하고 `useOpenViduSession`을 사용하여 WebRTC 연결을 구현해야 함
