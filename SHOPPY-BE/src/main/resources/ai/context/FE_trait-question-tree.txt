1. 질문 트리 소스
- trait-question-tree.json
- 이 JSON을 그대로 FE에 가져와서 “질문 → trait 자동매핑” 흐름으로 구성하면 됨.

2. 흐름 로직
- 시작 노드: startNodeId (q1)
- 각 노드에서 답변 선택 시:
    traitsAdd를 누적
    next로 이동
- next가 null이면 종료

3.질문/trait 매핑 (깨끗하게 정리)

q1: 조리를 할 수 있나요?
    예 → COOKING_OK
    아니오 → NO_COOKING
q2: 술이 포함되나요?
    예 → ALCOHOL_YES
    아니오 → ALCOHOL_NO
q3: 식사가 메인인가요?
    식사 → MEAL_MAIN
    간식/안주 → SNACK_MAIN
q4: 예산은 어떤 편인가요?
    가성비 → VALUE
    균형형 → BALANCED
    대용량 → BULK
q5: 장소는 어디인가요?
    실내 → INDOOR
    야외 → OUTDOOR
q6: 준비/정리 시간을 줄이고 싶나요?
    예 → MINIMAL
    아니오 → VARIETY_OK

4.카테고리와 연동(권장 룰)

- NO_COOKING 선택 시: MEAT_RAW, VEGETABLE_RAW, COOKING_TOOL 비활성
- ALCOHOL_NO 선택 시: ALCOHOL 비활성
- UI에 “카테고리 제한 적용됨” 표시

FE UI 컴포넌트 구조

구성 :

- src/features/room/questionnaire/
    QuestionFlow.tsx : 질문 흐름 제어(현재 노드, 답변 저장, 다음 노드 이동)
    QuestionCard.tsx : 질문 렌더링 카드(UI)
    QuestionOption.tsx : 선택지 버튼(YES/NO, CHOICE)
    traitQuestionTree.ts : trait-question-tree.json 파싱/정규화
    useQuestionFlow.ts : 상태 관리 훅(answers, traits 누적, next 계산)

역할 정리:

- QuestionFlow가 trait-question-tree.json 읽어서 질문 → traitsAdd → next 순으로 진행
- 마지막 노드 도달 시 traits 최종 배열 산출 → roomMeta.traits에 넣음
- 카테고리 선택은 별도 UI(체크박스), traits에 따라 비활성 처리
    NO_COOKING → MEAT_RAW / VEGETABLE_RAW / COOKING_TOOL 비활성
    ALCOHOL_NO → ALCOHOL 비활성
